StringeeX Django — Command & API Reference

Setup & initialization

setup

Install all Python dependencies

pip install -r requirements.txt
setup

Run database migrations (create or update tables)

python manage.py migrate
setup

Create a Django admin superuser (one-time)

python manage.py createsuperuser
setup

Generate a secure random access token for the dashboard

python -c "import secrets; print(secrets.token_urlsafe(24))"

Paste the output into ACCESS_TOKEN in .env

Data sync & management

data

One-shot live sync — fetches last 2 hours from StringeeX API

python manage.py sync_calls
data

One-shot sync — fetch last N hours from API

python manage.py sync_calls --hours 6

Replace 6 with any number of hours

data

Import from a downloaded CSV report file

python manage.py sync_calls --csv /path/to/call_report_20260508_20260508.csv
data

Backfill a date range from the StringeeX API

python manage.py sync_calls --start 2026-05-01 --end 2026-05-09

Fetches day by day. Historical records are never overwritten.

data

Wipe all records, then backfill from date range

python manage.py sync_calls --flush --start 2026-05-01 --end 2026-05-09
data

Flush (wipe) all call records — no re-sync

python manage.py sync_calls --flush

Destructive — deletes all rows from the database

data

Re-enrich team metadata on all existing records

python manage.py reenrich_teams

Run after updating Team_tradex.xlsx to apply new team mappings

Live daemon (continuous sync)

prod

Start the live sync daemon (every 4 minutes, recommended)

python manage.py sync_calls --daemon --interval 4

Today's records: update_or_create. Historical: get_or_create. Run in a separate terminal.

prod

Daemon with custom interval

python manage.py sync_calls --daemon --interval 10

Running the server

dev

Start development server (all interfaces, port 8000)

python manage.py runserver 0.0.0.0:8000
prod

Start production server with gunicorn (2 workers)

gunicorn stringeex_django.wsgi:application --bind 0.0.0.0:8000 --workers 2
prod

Collect static files for nginx/production

python manage.py collectstatic --noinput

Debugging & inspection

debug

Open Django shell for direct model queries

python manage.py shell

Then: from calls.models import CallRecord; CallRecord.objects.count()

debug

Check how many call records are in the database

python manage.py shell -c "from calls.models import CallRecord; print(CallRecord.objects.count())"
debug

Check the most recent call record ingested

python manage.py shell -c "from calls.models import CallRecord; r=CallRecord.objects.order_by('-start_time_ms').first(); print(r.start_time_ms, r.account_name)"
debug

List all Django management commands available

python manage.py help
debug

Check migration status (which have/haven't been applied)

python manage.py showmigrations
debug

Check for model changes not yet in migrations

python manage.py makemigrations --check --dry-run

API endpoints (test with curl or browser)

GET

KPIs for a single agent on a specific date

curl "http://localhost:8000/api/stats/?agent=email@domain.com&date=2026-05-13"
GET

Per-agent summary report (total, connected, talk time, queue time)

curl "http://localhost:8000/api/agent-summary/"

Optional filters: ?date=2026-05-13 ?tl=Rehan ?pool=Sales

GET

Raw call records with filters

curl "http://localhost:8000/api/calls/?connected=1&tl=Rehan"

Optional filters: ?connected=1 ?tl=Rehan ?date=2026-05-13

GET

Hourly call breakdown

curl "http://localhost:8000/api/hourly-breakdown/?date=2026-05-13"
GET

Team list (agents, TLs, pools)

curl "http://localhost:8000/api/team-list/"
GET

When was the last successful sync?

curl "http://localhost:8000/api/last-sync/"
POST

Hot-reload team data from Team_tradex.xlsx (no restart needed)

curl -X POST "http://localhost:8000/api/reload-team/"

Systemd services (production)

sys

Reload systemd after editing service files

sudo systemctl daemon-reload
sys

Start both web and sync services

sudo systemctl start stringeex-web stringeex-sync
sys

Check status of both services

sudo systemctl status stringeex-web stringeex-sync
sys

Restart both services (after code/config change)

sudo systemctl restart stringeex-web stringeex-sync
sys

Enable services to start on boot

sudo systemctl enable stringeex-web stringeex-sync
sys

Tail live logs from the sync daemon

sudo journalctl -u stringeex-sync -f
sys

Test nginx config and reload

sudo nginx -t && sudo systemctl reload nginx
